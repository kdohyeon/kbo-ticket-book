## 서비스 개요

**직관노트**는 KBO 야구 팬들이 직접 관람한 경기를 기록하고 관리하는 모바일/웹 앱이다.

- **플랫폼**: iOS, Android, Web (Expo + React Native)
- **인증**: Google, Apple, 카카오, 익명 로그인
- **데이터**: Firebase Firestore (실시간)
- **배포**: Vercel (Web), Expo (모바일)

## 백엔드 기능 스펙

현재 별도 서버 없이 Firebase 서비스를 직접 사용하는 BaaS(Backend as a Service) 구조다.

### 1. Firebase Authentication

| 기능 | 설명 |
|------|------|
| 소셜 로그인 제공자 | Google, Apple, Kakao (커스텀 토큰), Anonymous |
| 세션 관리 | Firebase SDK 자동 세션 유지 |
| 인증 상태 리스너 | `onAuthStateChanged`로 앱 전반 인증 상태 동기화 |

### 2. Firestore Database

**컬렉션 구조**

```
Firestore
├─ records/          # 사용자 직관 기록
│   └─ {recordId}    # 문서 (상세 필드는 앱 스펙 참조)
└─ games/            # KBO 경기 일정 (어드민 관리)
    └─ {gameId}
        ├─ date: string        # "YYYY-MM-DD"
        ├─ time: string        # "HH:mm"
        ├─ homeTeamId: string  # 팀 ID
        ├─ awayTeamId: string  # 팀 ID
        ├─ stadium: string     # 구장명
        ├─ homeScore?: number  # 홈팀 점수 (선택)
        ├─ awayScore?: number  # 원정팀 점수 (선택)
        └─ status: scheduled|playing|finished|canceled
```

**주요 쿼리 패턴**

| 용도 | 쿼리 |
|------|------|
| 내 기록 조회 | `records` WHERE `userId == uid` ORDER BY `createdAt DESC` |
| 경기 일정 조회 | `games` WHERE `date >= 월초` AND `date <= 월말` |
| 사용자 전체 조회 (어드민) | `records` (전체) GROUP BY `userId` |

**보안 규칙 (현재 구현 필요)**

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /records/{recordId} {
      allow read, write: if request.auth.uid == resource.data.userId;
      allow create: if request.auth != null;
    }
    match /games/{gameId} {
      allow read: if request.auth != null;
      allow write: if false; // 어드민만 Cloud Console에서 관리
    }
  }
}
```

### 3. Firebase Cloud Functions (카카오 로그인)

| 기능 | 엔드포인트 | 설명 |
|------|-----------|------|
| 카카오 커스텀 토큰 발급 | `POST /kakaoCustomToken` | 카카오 인가 코드 → Firebase 커스텀 토큰 교환 |

**요청/응답 구조**

```
Request:  { code: string, redirectUri: string }
Response: { firebaseToken: string }
```

## 인프라 설계

### 전제 조건

- 동시 사용자: **100명**
- 총 사용자: **5,000명**
- 서비스 특성: KBO 시즌 (3~11월) 집중, 경기일 저녁 트래픽 집중

### 현재 스택 평가

| 컴포넌트 | 현재 구성 | 평가 |
|---------|-----------|------|
| 앱 호스팅 | Vercel | 이 규모에 충분 |
| 인증 | Firebase Auth | 이 규모에 충분 |
| DB | Firestore | Blaze 플랜 전환 필요 |
| 스토리지 | Firebase Storage | 현재 미사용 |
| 서버리스 함수 | Firebase Functions | 이 규모에 충분 |
| 모바일 배포 | Expo EAS | 이 규모에 충분 |

### 권장 인프라 구성

#### Tier 1: Firebase (핵심 인프라)

**Firebase Blaze 플랜 (종량제) 전환 필수**

| 리소스 | 무료 한도 | 예상 사용량 | 예상 비용 |
|--------|-----------|------------|-----------|
| Firestore 읽기 | 50K/일 | ~200K/월 | ~$0.12/월 |
| Firestore 쓰기 | 20K/일 | ~50K/월 | ~$0.09/월 |
| Firestore 저장 | 1GB | < 100MB | $0 |
| Auth MAU | 10K 무료 | ~500/월 | $0 |
| Functions 호출 | 2M/월 무료 | ~10K/월 | $0 |
| **합계** | | | **~$1~5/월** |

> 시즌 경기일 저녁 피크 시 Firestore 읽기가 집중되지만, 5,000명 규모에서는 무료 한도 내 운영 가능성이 높음.

#### Tier 2: Vercel (웹 호스팅)

| 플랜 | 비용 | 적합성 |
|------|------|--------|
| Hobby (무료) | $0/월 | 초기 운영에 충분 |
| Pro | $20/월 | 팀 협업 기능 필요 시 |

- Bandwidth: 100GB/월 (Hobby) → 5,000명 규모에 충분
- 서버리스 함수 실행 시간 제한 없음 (정적 SPA이므로 해당 없음)
- 자동 CDN + HTTPS

#### Tier 3: Expo EAS (모바일 빌드/배포)

| 기능 | 비용 | 비고 |
|------|------|------|
| EAS Build | $0 (무료 30 빌드/월) | 초기 운영 충분 |
| OTA 업데이트 | $0 (무료 1K 업데이트/월) | |
| EAS Submit | 무료 | 스토어 자동 제출 |

### 비용 요약 (월 기준)

| 항목 | 비용 |
|------|------|
| Firebase Blaze | $1 ~ $5 |
| Vercel Hobby | $0 |
| Expo EAS | $0 |
| **총합** | **$1 ~ $5/월** |

> 총 사용자 5,000명, 동시 100명 수준에서는 현재 Firebase + Vercel 스택이 가장 비용 효율적이며 별도 서버 운영 불필요.

### 운영 시 주의사항

1. **Firestore 보안 규칙 설정 필수** — 현재 미설정 시 전체 공개 위험
2. **Firebase App Check 적용 권장** — 비정상 API 호출 차단
3. **어드민 역할 기반 접근 제어** — Custom Claims 또는 별도 컬렉션으로 어드민 계정 검증 추가
4. **Firestore 인덱스 최적화** — `userId + createdAt` 복합 인덱스 생성 권장
5. **Firebase 예산 알림 설정** — Blaze 플랜 과금 방지를 위한 월 $10 예산 알림 설정
